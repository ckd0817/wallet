# AGENTS.md

## 语言规则

- 本项目内新增的记忆文档、说明文档、产品文案建议，默认全部使用中文书写。
- 除非用户明确要求英文，或代码/协议字段本身必须保留英文，否则不要输出英文版说明。

## 本机环境记忆

- JDK 21 路径：`E:\Java\jdk-21`
- ADB 路径：`E:\AndroidStudioSDK\platform-tools\adb.exe`
- 这个工作区的终端会话有时会默认落到其他 JDK。
- 在执行 Gradle 构建、Android 测试、APK 打包前，先设置：
- `JAVA_HOME=E:\Java\jdk-21`
- 把 `E:\Java\jdk-21\bin` 放到 `PATH` 前面

## 安装发布记忆

- 只要本项目代码有改动，并且变更会影响 Android 实际运行结果，就默认重新打包并安装到我的手机上。
- 默认使用 `E:\AndroidStudioSDK\platform-tools\adb.exe` 安装。
- 默认覆盖安装到当前已连接并已授权的真机，不要每次额外询问。

## 产品文案记忆

- 不要默认把 UI、通知、按钮、提示文案写成解释性文字。
- 这是之前反复出现过的问题，尤其体现在：
- UI 设计文案
- 通知文案
- 最终文案要像正式产品，而不是像助手在解释逻辑。
- 优先使用简短、状态化、产品化的表达，不要堆说明。
- 示例方向：
- 推荐：`截图已完成` / `正在识别` / `识别完成`
- 避免：`用户可以放心离开当前页面` 这类解释性、指导性过强的补充话术，除非产品明确要求

## 产品呈现记忆

- 最终形态要以产品的形式呈现，不要带明显的解释痕迹。
- 设计 UI 时，能不写辅助说明就不写，只有在确实影响理解时才补充。
- 标签、空状态、横幅、通知、弹窗文案都要尽量简洁、明确、像成品。
